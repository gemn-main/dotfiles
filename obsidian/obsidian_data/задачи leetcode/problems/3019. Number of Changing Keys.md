```python
class Solution(object):
    def countKeyChanges(self, s):
        alf = {
        "a": 1, "A": 1,
        "b": 2, "B": 2,
        "c": 3, "C": 3,
        "d": 4, "D": 4,
        "e": 5, "E": 5,
        "f": 6, "F": 6,
        "g": 7, "G": 7,
        "h": 8, "H": 8,
        "i": 9, "I": 9,
        "j": 10, "J": 10,
        "k": 11, "K": 11,
        "l": 12, "L": 12,
        "m": 13, "M": 13,
        "n": 14, "N": 14,
        "o": 15, "O": 15,
        "p": 16, "P": 16,
        "q": 17, "Q": 17,
        "r": 18, "R": 18,
        "s": 19, "S": 19,
        "t": 20, "T": 20,
        "u": 21, "U": 21,
        "v": 22, "V": 22,
        "w": 23, "W": 23,
        "x": 24, "X": 24,
        "y": 25, "Y": 25,
        "z": 26, "Z": 26}
        o = 0
        for i in range(len(s)-1):
            if alf[s[i]] != alf[s[i+1]]:
                o += 1
        return o
```

Вам дана строка с индексом 0,s введенная пользователем. Изменение ключа определяется как использование ключа, отличного от последнего использованного ключа. Например, s = "ab"имеет изменение ключа, а s = "bBBb"не имеет.

Верните количество раз, которое пользователю пришлось сменить ключ.

Примечание: Модификаторы вроде shiftили caps lockне будут учитываться при изменении ключа, то есть если пользователь набрал букву 'a', а затем букву 'A', то это не будет считаться изменением ключа.

 

Пример 1:

Ввод: s = "aAbBcC"
 Вывод: 2
 Пояснение:  
От s[0] = 'a' до s[1] = 'A', нет смены клавиши, так как caps lock или shift не учитываются. 
От s[1] = 'A' до s[2] = 'b', есть смена клавиши. 
От s[2] = 'b' до s[3] = 'B', нет смены клавиши, так как caps lock или shift не учитываются. 
От s[3] = 'B' до s[4] = 'c', есть смена клавиши. 
От s[4] = 'c' до s[5] = 'C', нет смены клавиши, так как caps lock или shift не учитываются.

Пример 2:

Ввод: s = "AaAaAaaA"
 Вывод: 0
 Пояснение: Смена ключа не происходит, поскольку нажимаются только буквы «a» и «A» , что не требует смены ключа.
